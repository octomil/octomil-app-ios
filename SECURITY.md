# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

1. Email: **team@octomil.com**
2. Include a detailed description of the vulnerability
3. Steps to reproduce (if applicable)
4. Impact assessment

### What to Expect

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: We will evaluate the severity within 5 business days
- **Resolution**: Critical vulnerabilities will be patched within 14 days
- **Disclosure**: We follow coordinated disclosure practices

### Scope

- Octomil iOS App
- Device pairing and authentication flows
- Network communication with Octomil platform
- On-device model management and storage

### Out of Scope

- The Octomil iOS SDK itself (report to [octomil-ios](https://github.com/octomil/octomil-ios))
- Development tooling and CI/CD configs
- Third-party dependencies (report to upstream maintainers)
